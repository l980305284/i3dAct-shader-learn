### 第1篇: Shader是什么？为什么你要学它
**标题**: 《别被"着色器"这个名字吓跑——写给程序员的i3dAct Shader入门》

**核心目标**: 打破恐惧，建立直觉

**内容结构**:
1. **开篇悬念** - 一个简单需求：让一个3D球体每秒闪烁一次
   - 用传统方式：写脚本控制材质属性
   - 用Shader：5行代码搞定
2. **Shader本质** - 它不是魔法，就是"给每个像素写个函数"
   - 类比：像写一个 `function getColor(x, y) { return color; }`
   - 区别：GPU并行执行，同时跑百万次
3. **i3dAct Shader类型** - 先记住这俩：
   - `canvas_item` - 2D用的
   - `spatial` - 3D用的
4. **你的第一个Shader** - 让一个MeshRender变红
   ```glsl
   shader_type spatial;

   void fragment() {
       ALBEDO = vec3(1.0, 0.0, 0.0);
   }
   ```
5. **结尾钩子** - 下篇讲UV，让颜色"动起来"

**预期效果**: 读者能创建第一个shader文件，理解"fragment函数决定像素颜色"

### 第2篇: UV坐标——Shader世界的地图
**标题**: 《UV坐标：Shader世界的经纬度》

**核心目标**: 理解UV是什么，能用UV做渐变效果

**内容结构**:
1. **UV是什么** - 把模型表面"展开"到0-1的坐标系
   - 左下角(0,0)，右上角(1,1)
   - 可视化：把UV当颜色输出看看
2. **UV实战**:
   - 水平渐变: `ALBEDO = vec3(UV.x, 0.0, 0.0);`
   - 对角线渐变: 用UV.x和UV.y混合
   - 圆形渐变: 用distance函数
3. **TIME变量** - 让渐变动起来
   - `ALBEDO = vec3(UV.x + sin(TIME), ...)`
   - 呼吸灯效果
4. **小结** - UV就是"像素在模型表面的位置百分比"

**预期效果**: 读者能做出各种渐变效果，理解UV如何映射到像素

### 第3篇: 空间坐标系——Shader里的"我在哪"
**标题**: 《坐标系大乱斗：Model、World、View、Screen都是啥？》

**核心目标**: 理解各大坐标系的意义和转换

**内容结构**:
1. **为什么需要这么多坐标系** - 每个阶段有各自的"舒适区"
   - 模型自己关心"我的形状"
   - 世界关心"大家的位置关系"
   - 相机关心"我看到什么"
   - 屏幕关心"画在哪"
2. **四大坐标系详解**:
   | 坐标系 | 原点在哪 | 谁在用 | 典型用途 |
   |--------|---------|--------|---------|
   | Model（模型空间） | 模型中心 | 顶点数据 | 模型自身的形状 |
   | World（世界空间） | 场景原点 | 游戏逻辑 | 物体之间的位置关系 |
   | View（视图空间） | 相机位置 | 光照计算 | 相机看到什么 |
   | Screen（屏幕空间） | 屏幕左下角 | 最终渲染 | 像素画在哪 |
3. **i3dAct里的转换矩阵**:
   - `MODEL_MATRIX` - 模型 → 世界
   - `MODELVIEW_MATRIX` - 模型 → 视图
   - `INV_PROJECTION_MATRIX` - 投影的逆变换
4. **实战案例**: 让物体在世界空间里"晃动"
   ```glsl
   // 顶点着色器里做世界空间变换
   varying vec3 world_pos;
   void vertex() {
       world_pos = (MODEL_MATRIX * vec4(VERTEX, 1.0)).xyz;
   }
   ```
5. **常见坑**:
   - 分不清在哪个空间做计算
   - 法线变换要用专门的矩阵
   - UV不在任何空间里，它就是UV

**预期效果**: 读者理解为什么vertex函数里要乘矩阵，知道各空间的用途

### 第4篇: 纹理采样——不只是贴图
**标题**: 《texture()函数：Shader里最重要的那行代码》

**核心目标**: 理解如何读取纹理并处理

**内容结构**:
1. **为什么需要TEXTURE** - 你不可能用数学算出所有图案
2. **texture()函数** - 从纹理中"采样"
   ```glsl
   ALBEDO = texture(TEXTURE, UV).rgb;
   ```
3. **UV扭曲** - 让贴图"融化"
   - 给UV加偏移
   - 用sin制造波浪扭曲
4. **多纹理混合** - 渐变切换两张图
   ```glsl
   vec4 tex1 = texture(texture1, UV);
   vec4 tex2 = texture(texture2, UV);
   ALBEDO = mix(tex1, tex2, 0.5).rgb;
   ```
5. **实战案例**: 贴图切换过渡效果

**预期效果**: 读者能操作纹理，做出UV扭曲和贴图混合效果

### 第5篇: 数学函数——Shader的灵魂
**标题**: 《smoothstep、mix、clamp：Shader三剑客》

**核心目标**: 掌握最常用的数学工具

**内容结构**:
1. **smoothstep** - 平滑过渡的神器
   - 对比linear和smoothstep的区别
   - 用法：`smoothstep(边缘起点, 边缘终点, 值)`
2. **mix** - 线性插值，不只是混合颜色
   - 可以混合任何东西：颜色、坐标、透明度
3. **clamp** - 把值限制在范围内
   - 防止"越界"
4. **组合拳**: 溶解效果（Dissolve）
   - 用noise纹理 + smoothstep
   - 控制溶解进度
5. **延伸**: step函数和硬边缘

**预期效果**: 读者能独立实现溶解、边缘柔化等常见效果

### 第8篇: SDF模型剖切实战
**标题**: 《SDF入门：用数学切开你的模型》

**核心目标**: 理解SDF原理，实现模型剖切效果

**内容结构**:
1. **什么是SDF（Signed Distance Field）**:
   - 定义：空间中每一点到最近表面的"有向距离"
   - 正值 = 在物体外，负值 = 在物体内
   - 为什么SDF这么好用：光滑、可计算、易混合
2. **i3dAct里的SDF支持**:
   - CanvasItem的2D SDF: `texture_sdf()`, `texture_sdf_normal()`
   - 3D SDF需要自己写或用预计算纹理
3. **基础SDF形状**（手写公式）:
   ```glsl
   // 球体SDF
   float sdf_sphere(vec3 p, float r) {
       return length(p) - r;
   }
   // 盒子SDF
   float sdf_box(vec3 p, vec3 b) {
       vec3 q = abs(p) - b;
       return length(max(q, 0.0)) + min(max(q.x, max(q.y, q.z)), 0.0);
   }
   ```
4. **实战案例：模型剖切效果**:
   - 用平面方程切割模型
   - 显示内部结构（可选：不同颜色）
   - 添加剖切边缘发光
   ```glsl
   // 简单平面剖切
   float clip_plane = dot(world_pos, plane_normal) - plane_distance;
   if (clip_plane < 0.0) {
       discard; // 或显示内部颜色
   }
   ```
5. **进阶：SDF动画**:
   - 动态剖切面
   - 多个SDF布尔运算（并、交、差）
   - 边缘软化
6. **系列总结** - 学完这8篇，你已经能做很多事了

**预期效果**: 读者能理解SDF原理，实现基础的模型剖切效果
