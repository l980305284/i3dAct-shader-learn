# 第13篇：全屏 Shader 入门 设计文档

**日期**: 2026-06-27
**文件**: `articles/13-fullscreen-shader-intro.md`
**标题**: 《全屏 Shader 入门：把整张画面当成一张纹理来处理》

## 定位

系列三（后处理/全屏效果）的开篇。第8-12篇一直围绕模型剖切、SDF 和动画展开，本篇切换到新的思路：不再处理单个模型表面，而是把整张屏幕当作一张纹理读取、修改、再输出。

本篇的目标不是做复杂滤镜，而是让读者建立全屏后处理的基本模型：

```text
读取屏幕原颜色 -> 修改颜色 -> 输出到当前像素
```

## 目标读者

- 已读完前12篇，理解 `UV`、`texture()`、`SCREEN_UV` 的基础概念
- 会写 `fragment()`，知道 shader 是逐像素运行
- 第一次正式接触全屏 Shader / 后处理

## 写作风格

- 代码先行，后解释
- 尽量使用一个完整主代码块，避免拆成很多小代码段
- 像教同事一样解释，不使用 AI 模板句
- 强调“换了处理对象”：从模型贴图换成屏幕画面
- 不引入模糊、卷积、深度纹理等后续主题，避免第13篇变重

## 主代码设计

文章主体使用一个完整的全屏 shader 代码块，包含以下模式：

- `effect_mode = 0`: 原图
- `effect_mode = 1`: 灰度
- `effect_mode = 2`: 反色
- `effect_mode = 3`: 亮度调整
- `effect_mode = 4`: 对比度调整

主代码块重点展示完整后处理框架：

- 声明 `shader_type canvas_item`
- 声明 `effect_mode`、`brightness`、`contrast`
- 使用 `texture(SCREEN_TEXTURE, SCREEN_UV)` 读取屏幕颜色
- 根据模式修改颜色
- 输出最终颜色

后文讲解只引用关键变量和关键行，不反复贴碎片代码。

## 内容结构

### 1. 开篇：从模型 Shader 切到全屏 Shader

- 承接第12篇：之前切模型、改模型表面，现在改整张画面
- 用一句话定义全屏 Shader：把已经渲染好的屏幕画面再处理一遍
- 说明本篇先做基础效果：原图、灰度、反色、亮度、对比度

### 2. 先给完整代码

- 放一个完整可复制的全屏 shader
- 代码中保留必要注释
- 不在代码前堆太多概念，先让读者看到完整形态

### 3. 解释这份代码在做什么

- `SCREEN_TEXTURE`：当前屏幕画面
- `SCREEN_UV`：屏幕上的 0-1 坐标
- `texture(SCREEN_TEXTURE, SCREEN_UV)`：读取当前像素原本的屏幕颜色
- `COLOR = vec4(final_color, screen_color.a)`：把处理后的颜色写回去

核心比喻：以前是“拿模型 UV 去读模型贴图”，现在是“拿屏幕 UV 去读屏幕画面”。

### 4. 五种模式分别说明

- 原图：验证屏幕采样是否正常
- 灰度：用加权灰度，不简单取 `r`
- 反色：`1.0 - color`
- 亮度：整体加减颜色值，并用 `clamp` 限制范围
- 对比度：围绕 `0.5` 拉开或压缩颜色

每种模式主要用文字解释，不再单独拆小代码块。

### 5. 常见坑

- 用模型 `UV` 采样屏幕，导致结果不符合预期
- 一上来做复杂效果，结果不知道是采样错还是算法错
- 灰度直接取红色通道，画面偏差明显
- 调亮度/对比度后忘记 `clamp`，导致颜色越界

### 6. 这篇你学到了

- 全屏 Shader 的核心流程：读屏幕、改颜色、输出
- `SCREEN_TEXTURE` 和 `SCREEN_UV` 的作用
- 一个基础后处理框架可以切换多种简单效果
- 后续颜色滤镜、屏幕扭曲、暗角、像素化都基于这个框架扩展

### 7. 下篇预告

第14篇进入颜色滤镜：不只是灰度和反色，而是用颜色矩阵、通道混合、冷暖色调做更可控的画面风格。
